# 文档四：Qt 前端界面设计思路

## 1. 技术选型
- Qt 版本：6.x
- 主框架：Qt Widgets（用于配置管理、表格编辑、树形控件等）
- 图表与监控：QML + ECharts（通过 `QQuickWidget` 嵌入），支持热更新
- 网络通信：`QNetworkAccessManager`（HTTP）、`QWebSocket`（实时推送）
- 本地缓存：JSON 文件（存储当前配置快照，用于快速启动和离线使用）

## 3. 核心界面模块
### 3.1 配置管理界面（Widgets）
- 采集点表：使用 `QTableView` 展示，支持内联编辑、排序、过滤。每行代表一个采集点，包含名称、协议类型、寄存器地址等。
- 数据模块表：展示数据处理模块（如标度变换、滤波），支持编辑参数（JSON 格式）。
- 转发规则表：展示对外映射规则，支持拖拽或下拉选择关联的数据模块和采集点。
- 树形导航：左侧使用 `QTreeView`，按“设备 -> 模块 -> 采集点 -> 转发规则”组织，提供快捷筛选。

### 3.2 图表监控界面（QML + WebView）
- 用于展示实时数据曲线、设备状态等，界面可热更新。
- 使用 `QQuickWidget` 加载 QML 文件，通过 `QWebChannel` 将 C++ 数据模型暴露给 JavaScript，实现动态图表更新。
- QML 资源包从后端下载并支持版本管理。

### 3.3 批量编辑与向导
- 批量修改：支持选中多行后右键批量修改（如统一修改寄存器地址、分配模块）。
- 向导式创建：引导用户三步完成“添加采集点 -> 绑定数据模块 -> 设置转发规则”。

### 3.4 冲突处理对话框
- 当收到 `409` 冲突响应时，自动弹出对话框，显示最新值与用户修改值对比，用户可选择覆盖或放弃。

## 4. 数据流与状态管理
### 4.1 启动流程
1. 读取本地缓存 JSON（若存在且未超过 1 小时），快速显示上次配置。
2. 同时检测后端服务是否可用（`GET /api/version`）。
3. 若后端可用，建立 WebSocket 连接并拉取全量最新配置，与本地缓存合并（以后端为准）。
4. 检查 QML 版本，如有新版本则下载并提示刷新图表区域。

### 4.2 编辑与保存（乐观更新）
1. 用户在表格中修改单元格后，立即更新本地模型和 UI。
2. 同时将更新请求入队，调用 `PUT /api/points/{id}`。
3. 后端返回成功：无需额外动作。
4. 后端返回 `409`：自动拉取最新数据，回滚该单元格，并弹出冲突提示。

### 4.3 实时同步（WebSocket 处理）
- 收到 `config_changed` 推送后，判断当前视图是否显示该表。
- 若是，按推送内容重拉对应记录或整表。
- 若为删除操作，直接从模型和视图中移除该项。

### 4.4 离线模式与重连
- WebSocket 断开时，界面显示“离线”标识，并禁用编辑（或允许编辑但缓存操作）。
- 周期性重连，恢复后执行增量同步或全量同步。

## 5. 用户交互优化细节
- 即时校验：输入寄存器地址时校验格式与重复性，实时提示错误。
- 拖拽分配：将采集点从左侧树拖拽到数据模块节点，自动更新采集点的 `module_id`。
- 撤销/重做：基于 `QUndoStack`，每次修改生成命令，支持撤销到上一步。
- 快捷筛选：表格上方提供过滤器（如按协议类型、数据模块筛选）。
- 配置导入/导出：支持导出为 JSON 或 Excel，便于备份和迁移。

## 6. QML 热更新机制
- 版本检查：主窗口启动后定时（如每小时）或手动调用 `GET /api/qml/version`。
- 下载与替换：若版本号不同，下载 ZIP 到临时目录，解压后替换本地 `qml/` 目录。
- 刷新图表：清空 `QQuickWidget` 引擎缓存，重新加载 `main.qml`。
- 降级：若下载失败或新 QML 加载错误，自动回退到上一版本。

## 7. 性能考量
- 大数据量表格：使用 `QAbstractTableModel`，配合分页/批量加载，避免一次性加载全部数据。
- 图表实时刷新：通过 WebSocket 推送增量数据，只更新变化的数据点，避免整图重绘。
- 懒加载：树形导航仅在展开节点时加载子节点内容。

## 8. 与后端通信封装
- 创建 `ApiClient` 类封装所有 HTTP 请求，使用异步回调或 future/promise 处理响应。
- 创建 `WebSocketClient` 类，处理连接、重连、消息解析。
- 使用 `QSettings` 存储后端地址、端口等配置。

## 9. 界面风格与主题
- 支持深色/浅色主题切换（基于 Qt 样式表）。
- 工业风格控件（扁平化、高对比度）。

## 10. 测试与调试
- 提供模拟模式：不连接后端，使用本地假数据进行界面测试。
- 提供独立日志窗口，显示 API 请求/响应和 WebSocket 消息，便于排查问题。
